# Features 模块说明

本目录包含所有业务功能模块，采用功能导向的组织结构。

## 目录结构

```
features/
├── core/              # 核心功能模块
│   ├── auth/         # 用户认证与授权
│   ├── dashboard/    # 仪表板与概览
│   └── settings/     # 系统设置
├── chat/             # 聊天功能
│   ├── ai-chat/     # AI对话功能
│   └── conversations/ # 用户对话管理
├── documents/        # 文档管理
├── knowledge/        # 知识管理
├── users/           # 用户管理
└── tasks/           # 任务管理
```

## 模块设计原则

### 1. 自包含原则
每个功能模块应该是自包含的，包含：
- `components/` - 该功能专用的组件
- `hooks/` - 该功能专用的hooks
- `services/` - 该功能的API服务
- `types/` - 该功能的类型定义
- `utils/` - 该功能的工具函数
- `index.tsx` - 功能模块的主入口

### 2. 最小依赖原则
- 模块间应避免直接依赖，通过共享层进行交互
- 使用 `/shared` 目录中的组件和工具
- 通过状态管理进行数据共享

### 3. 一致性原则
- 所有模块遵循相同的目录结构
- 使用统一的命名约定
- 遵循统一的代码风格

## 使用指南

### 创建新功能模块
1. 在对应分类下创建新目录
2. 创建标准的目录结构
3. 实现功能组件和逻辑
4. 更新路由配置
5. 添加模块文档

### 模块间通信
- 使用全局状态管理（Zustand）
- 通过事件系统进行松耦合通信
- 使用共享的API服务层

### 测试策略
- 每个模块包含对应的测试文件
- 单元测试覆盖核心逻辑
- 集成测试验证模块交互
